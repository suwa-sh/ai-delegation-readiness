# CLAUDE.md

## このリポジトリの正体

**ai-delegation-readiness** — 高リスクな定型業務を AI に委任するための 4 層前提条件 +
監査ログ設計テンプレートの **ドキュメント中心の参照実装**。コード実装言語は持たない。
MIT の OSS。

味の素グループ(AFS)の経理 AI エージェント事例から、再現可能な骨格(4 層フレーム +
監査ログ最小スキーマ + 委任マトリクス)を抽出し、自社で同種の委任を検討する組織が
雛形として持ち帰れるように整えるのが目的。

## 編集規約

- **コメント・本文は日本語**(タイトル・識別子・コード断片の言語は対象に合わせる)
- 図は **mermaid** で書き、追加・変更したら `npx md-mermaid-lint <ファイル>` で検証する
- 各 doc の主張は **【観測事実】**(味の素事例の公開情報から確認できた内容)と
  **【設計提案】**(本リポでの一般化・補完)を明示的にラベル分けする。事実と推論の境界を
  曖昧にしない
- 章立てを変えるときは 4 層フレーム(`docs/01`)との対応関係が崩れないか確認する

## 更新運用

- 味の素事例の④統制層は公開情報が薄い。続報(誤承認補正フロー・人間最終承認の置き場所・
  規定バージョン管理運用)が出た時点で `docs/01` `docs/02` を更新する
- 監査ログの「拡張点」(改ざん耐性・保存期間・原証憑参照・規定バージョン固定)は
  読者から実装提案が来た時点で例示を追加する
- agent-loop 点検メモ(`docs/04`)は本リポ作者の自社運用例。他者環境への当てはめは
  各自で再点検する

## 横断的な注意点

- **GitHub 公開はユーザー判断**で行う(MVP では `gh repo create` しない)
- LICENSE は MIT。秘密情報・ハードコード認証情報の混入なきこと
- README の売り(「ドキュメント中心の参照実装」「コード実装言語なし」)と本文の整合を保つ
